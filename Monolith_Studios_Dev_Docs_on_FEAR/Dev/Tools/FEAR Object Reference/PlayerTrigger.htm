PlayerTrigger
PlayerTrigger objects are Triggers that will activate only when all players in a level are within the trigger. Used for multiplayer.

Message Description Example
VISIBLE <bool> Toggles whether the object is visible. msg <ObjectName> (VISIBLE 1)<BR>msg <ObjectName> (VISIBLE 0)
SOLID <bool> Toggles whether the object is solid. msg <ObjectName> (SOLID 1)<BR>msg <ObjectName> (SOLID 0)
HIDDEN <bool> Toggles whether the object is visible, solid, and rayhit. msg <ObjectName> (HIDDEN 1)<BR>msg <ObjectName> (HIDDEN 0)
REMOVE Removes the object from the game permanently. msg <ObjectName> REMOVE
CASTSHADOW <bool> Toggles whether the object casts a shadow. msg <ObjectName> (CASTSHADOW 1)<BR>msg <ObjectName> (CASTSHADOW 0)
SETPOS <X> <Y> <Z> DEBUGGING ONLY. Sets the position of an object. msg <ObjectName> (MOVETOPOS 50.5 100 75)
MOVETOPOS <X> <Y> <Z> DEBUGGING ONLY. Moves to the position, with collisions. msg <ObjectName> (MOVETOPOS 50.5 100 75)
SETROTATION <yaw> <pitch> <roll> DEBUGGING ONLY. Rotates object to euler angles. msg <ObjectName> (SETROTATION 90.0 0 0)
COPYXFORM <object> DEBUGGING ONLY. Copies trasform of <object> to this object. msg <ObjectName> (COPYXFORM ThePoint)
LOCK Locks the Trigger so that it ignores all commands other than UNLOCK and REMOVE and cannot be activated by the player msg Trigger LOCK
UNLOCK Unlocks the Trigger msg Trigger UNLOCK
ON Tells the object to send its commands msg Trigger ON
TEAM <0, 1, -1> Specifies which team can activate this trigger msg Trigger (TEAM 1)

Dims The dimensions of the trigger in the game. If the player has to walk into the trigger to fire it, you should set the dimensions. If the trigger is fired off by another object, then they don't need to be changed. Trigger dimensions are set as x, y and z values. You can make each of them as small as 0 or as large as you need. For example, for a trigger that fills a typical doorway you would set the dims to 128 128 4. Using the 4 is just to make the object easier to find. Triggers can be any size.
