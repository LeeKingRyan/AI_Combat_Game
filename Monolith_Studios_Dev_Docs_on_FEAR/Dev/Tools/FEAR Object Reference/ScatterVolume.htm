ScatterVolume
A scatter volume will generate a collection of particles on the geometry contained within the dimensions of the object and will handle rendering these particles at runtime.

Message Description Example
VISIBLE <bool> Toggles whether the object is visible. msg <ObjectName> (VISIBLE 1)<BR>msg <ObjectName> (VISIBLE 0)
SOLID <bool> Toggles whether the object is solid. msg <ObjectName> (SOLID 1)<BR>msg <ObjectName> (SOLID 0)
HIDDEN <bool> Toggles whether the object is visible, solid, and rayhit. msg <ObjectName> (HIDDEN 1)<BR>msg <ObjectName> (HIDDEN 0)
REMOVE Removes the object from the game permanently. msg <ObjectName> REMOVE
CASTSHADOW <bool> Toggles whether the object casts a shadow. msg <ObjectName> (CASTSHADOW 1)<BR>msg <ObjectName> (CASTSHADOW 0)
SETPOS <X> <Y> <Z> DEBUGGING ONLY. Sets the position of an object. msg <ObjectName> (MOVETOPOS 50.5 100 75)
MOVETOPOS <X> <Y> <Z> DEBUGGING ONLY. Moves to the position, with collisions. msg <ObjectName> (MOVETOPOS 50.5 100 75)
SETROTATION <yaw> <pitch> <roll> DEBUGGING ONLY. Rotates object to euler angles. msg <ObjectName> (SETROTATION 90.0 0 0)
COPYXFORM <object> DEBUGGING ONLY. Copies trasform of <object> to this object. msg <ObjectName> (COPYXFORM ThePoint)


Property Description Type Default
Name The name of this object String noname
Pos The position in the world of this object. Vector (Distance) 0.000000, 0.000000, 0.000000
Rotation The orientation of this object represented by euler angles in degrees Rotation 0.000000, 0.000000, 0.000000
Template Indicates that this object will not exist at runtime, and may be used as an object template for Spawner objects Boolean 0.000000
IsGore Indicates that this object is gory and should not exist for low-violence clients. Boolean 0.000000
ObjectLOD This value indicates at which detail levels the object will exist. For example at Low, this object will always exist, at Medium it will only exist in medium and high detail, and Never will cause the object to never exist. String (StaticList) Low
Dims The Dims define the volume an effect takes place in. It is an area defined in WorldEdit units as a box around the center of the effect object. Vector (LocalDims) 128.000000, 128.000000, 128.000000
Density The number of particles placed per 256 square units of surface area. The density can be reduced by placement material blending and clumpiness. Float 128.000000
Clumpiness A number between 0 and 1 that specifies how much particles will 'clump' together. 0 distributes particles evenly over the surface, while 1 will result in very clumpy placement. This works about the same as a 'Noise' material in Max or Maya. Float 0.000000
ClumpSize The approximate dimensions of a clump before it has been modified with the clump cutoff value. Also the approximate distance between clumps. Float 256.000000
ClumpCutoff A number between 0 and 1 that specifies how sharp the edges of clumps should be. 0 will result in a smooth falloff in density towards the edges of the clump, while higher numbers will result in sharper edges and fewer stragglers. Numbers above about 0.7 will result in VERY sparse placement. Float 0.500000
ClumpOffset By default, different scatter volumes will have clumps in the same locations in world space. This can be useful for having multiple materials within the same clump. If you really want the clumps to be independent of each other, change the clump offset in one of the volumes. A good value might be 50 50 50 for example. Vector 0.000000, 0.000000, 0.000000
Width The base width of the quad that is rendered for each particle. Note that this is independent of material aspect ratio, so if you material is 256x64, you would probably enter a width that is 4 times larger than the height. Float 64.000000
Height The base height of the quad that is rendered for each particle. Note that this is independent of material aspect ratio, so if you material is 256x64, you would probably enter a width that is 4 times larger than the height. Float 64.000000
PlantDepth A value that specifies how far the particle should be embedded in the surface. If this is negative, then the particles will float above the surface. Float 0.000000
Tilt The range of tilt that particles may be placed at. This value is in degrees. Float 30.000000
ScaleRange The +/- range of scales that different particles will have. For example, value of 0.2 will result in particles that are between 80%% and 120%% of the base width and height. Float 0.200000
WaveRate The speed of the wave motion of the particles. This is in degrees per second, so 360 will result in a complete wave cycle every second, while 90 will result in a complete wave cycle every 4 seconds. Float 90.000000
WaveDist The distance in WorldEdit units to move the top of the particles. For example, value of 10 will result in a total movement range of 10 units in either direction. Float 10.000000
WaveSize The approximate dimensions of an area of common movement. Over this area, particles will wave approximately in unison, smoothly blending to other nearby motions. This works about the same as a 'Noise' texture in Max or Maya. Float 128.000000
WaveDirSize The size of areas of similar wind direction. Float 128.000000
MaxDrawDist The maximum distance at which particles in a subvolume of this volume will be visible. Float 1024.000000
MaterialName The material that will be placed on the rendered quad. Particles are placed on the surface based on the bottom center of the quad, so it is usually best for materials to have an anchor point at the bottom center as well. String (Filename)
PlacementMaterial This is an optional material that is only used for placing materials. If this is not empty, then particles will only be placed on polygons that have this material applied to them. If multi-texturing is in use, then the amount of this material will be used to change the density based on vertex alpha. String (Filename)
PlacementCutoff If multi-texturing is in use on a polygon, then points on the surface with a vertex alpha value less than this value will not have any particles placed on them. Float 0.500000
BaseColor The base color of the particles before lighting is added. Color 255.000000, 255.000000, 255.000000
TranslucentLight No lights contribute to the color of the particles, only the base color is used. Boolean 0.000000
Translucent Whether or not the particles that this scatter volume creates are translucent or not. Boolean 1.000000
BackFaces This determines if a back face should be rendered for the scatter particles. This should ONLY be set when using a one sided shader as it slows down rendering Boolean 0.000000
NumImages This indicates the number of images stored in the material. These should be layed out like an image strip along the X axis, so if there are two images, image one sould go from 0-.5, and image two should go from .5-1 horizontally. Float 1.000000
