KeyFramer
The keyframer is a class that will move the specified objects along a path and can trigger sounds and commands as it moves. The path is specified using either key objects or a path stored in an external key file. The movement rate can either be specified using key times, or can use the distance between keys if TotalPathTime is set to non-zero, in which case it will traverse the full path in that specified amount of time.

VISIBLE <bool> Toggles whether the object is visible. msg <ObjectName> (VISIBLE 1)<BR>msg <ObjectName> (VISIBLE 0)
SOLID <bool> Toggles whether the object is solid. msg <ObjectName> (SOLID 1)<BR>msg <ObjectName> (SOLID 0)
HIDDEN <bool> Toggles whether the object is visible, solid, and rayhit. msg <ObjectName> (HIDDEN 1)<BR>msg <ObjectName> (HIDDEN 0)
REMOVE Removes the object from the game permanently. msg <ObjectName> REMOVE
CASTSHADOW <bool> Toggles whether the object casts a shadow. msg <ObjectName> (CASTSHADOW 1)<BR>msg <ObjectName> (CASTSHADOW 0)
SETPOS <X> <Y> <Z> DEBUGGING ONLY. Sets the position of an object. msg <ObjectName> (MOVETOPOS 50.5 100 75)
MOVETOPOS <X> <Y> <Z> DEBUGGING ONLY. Moves to the position, with collisions. msg <ObjectName> (MOVETOPOS 50.5 100 75)
SETROTATION <yaw> <pitch> <roll> DEBUGGING ONLY. Rotates object to euler angles. msg <ObjectName> (SETROTATION 90.0 0 0)
COPYXFORM <object> DEBUGGING ONLY. Copies trasform of <object> to this object. msg <ObjectName> (COPYXFORM ThePoint)
ON Tells the KeyFramer to start moving its listed objects along the path defined by its specified keys starting on the 0 key msg KeyFramer ON
OFF Tells the KeyFramer to stop moving its listed objects along the path defined by its specified keys. The objects stop wherever they are on the path msg KeyFramer OFF
PAUSE Tells the KeyFramer to pause the movement of its listed objects along the path defined by its specified keys msg KeyFramer PAUSE
RESUME Tells the KeyFramer to pause the movement of its listed objects along the path defined by its specified keys msg KeyFramer RESUME
FORWARD Tells the KeyFramer to move its listed objects along the path defined by its specified keys in a forward direction msg KeyFramer FORWARD
REVERSE Tells the KeyFramer to move its listed objects along the path defined by its specified keys in a backward direction msg KeyFramer REVERSE
TOGGLEDIR Tells the KeyFramer to move its listed objects along the path defined by its specified keys in the opposite direction it is currently traveling in msg KeyFramer TOGGLEDIR
GOTO <key> Tells the KeyFramer to immediately move its listed objects to a specific Key along the path defined by its specified keys To tell a set of objects Keyframed by a KeyFramer named "KeyFramer" to goto a specific Key named Key03 the command would look like:

msg KeyFramer (GOTO Key03)
MOVETO <key> [destination cmd] Tells the KeyFramer to move its listed objects to a specific Key along the path defined by its specified keys by traveling the path to that specific Key. It will also send the command listed in the optional �destion cmd� paramater To tell a set of objects Keyframed by a KeyFramer named "KeyFramer" to move to a specific Key named Key03 the command would look like:

TARGET <object> Tells the KeyFramer to rotate the Z vector of its listed object toward a specific target object To tell a set of objects Keyframed by a KeyFramer named "KeyFramer" to face the player the command would look like:

msg KeyFramer (TARGET player)
CLEARTARGET Tells the KeyFramer to stop aligning the Z vector of its specified objects to face a specified target msg KeyFramer CLEARTARGET
TARGETOFFSET <x> <y> <z> If a target name is specified, KeyFramed objects will always face the targeted object�s position plus the offset specified in the target offset parameter To tell a set of objects Keyframed by a KeyFramer named "KeyFramer" that is targeting the player to maintain an offset 5, 5, and 10 the command would look like:

Property Description Type Default
Name The name of this object String noname
Pos The position in the world of this object. Vector (Distance) 0.000000, 0.000000, 0.000000
Rotation The orientation of this object represented by euler angles in degrees Rotation 0.000000, 0.000000, 0.000000
Template Indicates that this object will not exist at runtime, and may be used as an object template for Spawner objects Boolean 0.000000
IsGore Indicates that this object is gory and should not exist for low-violence clients. Boolean 0.000000
ObjectLOD This value indicates at which detail levels the object will exist. For example at Low, this object will always exist, at Medium it will only exist in medium and high detail, and Never will cause the object to never exist. String (StaticList) Low
ObjectName This feild contains the name of the object or worldmodel that is to be affected by the keyframer. This feild can contain the names of several objects and/or worldmodels seperated by semicolons. String
Interpolation This indicates how movement between keys will be handled. Linear is much cheaper but doesn't look as smooth for a small number of keys. Bezier is more expensive but smoother and requires tangents. String (StaticList) Linear
BaseKeyName The base name of the keys that the KeyFramer object will follow. The first key must be named with the base name followed by 0. All subsequent keys must be named with the base name followed by two-digit increments. For example, if the BaseKeyName was 'Base', the keys would be named Base0, Base01, Base02, Base03, etc. String
ExternalKeyFile If your animation keys are in a separate .Key00c file, exported from max, enter the file path of your animations here. The KeyFramer will then look at the file for the keys. String
PushObjects Determines whether or not the KeyFramer will push objects in the world. Boolean 0.000000
StartActive If set to true, the KeyFramer will begin active. Boolean 0.000000
StartPaused This flag works in conjunction with the StartActive flag. This flag will tell the KeyFramer to begin in the paused state if active. Boolean 0.000000
Looping If set to true the keyframer will continuously run through the key path, returning to the first key in the sequence when reaching the last. Boolean 0.000000
MaintainOffsets Tells the KeyFramer whether or not to maintain offsets of position and rotation between the object being keyframed and the keys that make up the path. If set to false, the keyframed object will acquire the exact position and rotation of the keys that make up the key path. Boolean 1.000000
TargetName You can specify a prop, worldmodel, or model in this field. The KeyFramer will attempt to continually point it's forward facing vector at this object as it travels along its key path. String
TargetOffset You may enter a target offset in this field. It is entered in the form of x,y,z offset in WorldEdit units. Vector 0.000000, 0.000000, 0.000000
ActiveSound The name of a sound record that will be played while the KeyFramer object is active. String (StaticList)
Waveform The WaveForm flag is a subset of properties that define the movement characteristics of the KeyFramer object. Integer (GroupOwner) 0.000000
TotalPathTime This is the approximate time in seconds that it will take for the KeyFramer object to complete it's travel down the key path. Note that by specifying this, it will ignore the time stamps of the keys and instead use the distance between the keys as the primary factor Float 0.000000
Wavetype This is a dropdown list of different Waves that the KeyFramer can use while following it's path. The KeyFramer will vary it's velocity depending on the wave type chosen. Note that this is only used if the TotalPathTime is non-zero, otherwise it is ignored String (StaticList) Linear
GlobalResources Indicates whether the object's resources should be considered as global in the resource streaming system. Boolean 0.000000
