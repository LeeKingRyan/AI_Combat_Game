SnowVolume
SnowVolumes are used to define a spatial area where snow particles will be created and fall towards the ground.

Message Description Example
VISIBLE <bool> Toggles whether the object is visible. msg <ObjectName> (VISIBLE 1)<BR>msg <ObjectName> (VISIBLE 0)
SOLID <bool> Toggles whether the object is solid. msg <ObjectName> (SOLID 1)<BR>msg <ObjectName> (SOLID 0)
HIDDEN <bool> Toggles whether the object is visible, solid, and rayhit. msg <ObjectName> (HIDDEN 1)<BR>msg <ObjectName> (HIDDEN 0)
REMOVE Removes the object from the game permanently. msg <ObjectName> REMOVE
CASTSHADOW <bool> Toggles whether the object casts a shadow. msg <ObjectName> (CASTSHADOW 1)<BR>msg <ObjectName> (CASTSHADOW 0)
SETPOS <X> <Y> <Z> DEBUGGING ONLY. Sets the position of an object. msg <ObjectName> (MOVETOPOS 50.5 100 75)
MOVETOPOS <X> <Y> <Z> DEBUGGING ONLY. Moves to the position, with collisions. msg <ObjectName> (MOVETOPOS 50.5 100 75)
SETROTATION <yaw> <pitch> <roll> DEBUGGING ONLY. Rotates object to euler angles. msg <ObjectName> (SETROTATION 90.0 0 0)
COPYXFORM <object> DEBUGGING ONLY. Copies trasform of <object> to this object. msg <ObjectName> (COPYXFORM ThePoint)
ON This message turns the snow effect on. The effect will turn on through precipitation of particles from the top of the volume msg SnowVolume ON
OFF This message turns the particle system off. The particles disappear immediately msg SnowVolume OFF

Name The name of this object String noname
Pos The position in the world of this object. Vector (Distance) 0.000000, 0.000000, 0.000000

ObjectLOD This value indicates at which detail levels the object will exist. For example at Low, this object will always exist, at Medium it will only exist in medium and high detail, and Never will cause the object to never exist. String (StaticList) Low
Dims The Dims define the volume an effect takes place in. It is an area defined in WorldEdit units as a box around the center of the effect object. Vector (Dims) 128.000000, 128.000000, 128.000000
Density The number of particles placed per 256 cubic units of volume. Snow volumes with different dimensions (vertical and XZ) but with the same density can be seemlessly placed next to each other since the density is relative to the volume and doesn't represent a specific number of particles. Float 512.000000
ParticleRadius The approximate radius of a particle in WorldEdit units. Float 1.000000
FallRate The speed at which snow will fall in WorldEdit units per second. A value of 0 will result in particles that don't fall, but may still spin. Negative values will not generate desirable results. Float 80.000000
TumbleRate The number of degrees of rotation per second around a central point. Float 180.000000
TumbleRadius The distance from the base particle position to rotate at. 0 will result in particles that fall straight down, 10 will result in particles that flutter up to 10 units in either direction as they fall. Float 5.000000
MaxDrawDist The maximum distance at which particles are drawn. Starting at about 80%% of this distance, particles will begin to fade out. Float 1024.000000
BaseColor The base color of the particles before lighting is added. Color 255.000000, 255.000000, 255.000000
TranslucentLight No lights contribute to the color of the particles, only the base color is used. Boolean 0.000000
Translucent Whether or not the particles that this scatter volume creates are translucent or not. Boolean 1.000000
BackFaces This determines if a back face should be rendered for the scatter particles. This should ONLY be set when using a one sided shader as it slows down rendering Boolean 0.000000
MaterialName The material to render the particles with. All of the mip levels need to have a black border at the top and right edges of the material in order to render correctly. The material needs to use non-premultiplied alpha (for example, entirely white color channel, with the actual shape in the alpha channel.) String (Filename)
StartOn Should the SnowVolume start emitting snow when it is created? Boolean 1.000000
