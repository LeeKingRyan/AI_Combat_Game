WorldModel
WorldModels are used to specify dynamic geomotry in the world that may be keyframed, have physics properties, be destroyed or interact in other ways with the player or other objects in the world.

Message Description Example
VISIBLE <bool> Toggles whether the object is visible. msg <ObjectName> (VISIBLE 1)<BR>msg <ObjectName> (VISIBLE 0)
SOLID <bool> Toggles whether the object is solid. msg <ObjectName> (SOLID 1)<BR>msg <ObjectName> (SOLID 0)
HIDDEN <bool> Toggles whether the object is visible, solid, and rayhit. msg <ObjectName> (HIDDEN 1)<BR>msg <ObjectName> (HIDDEN 0)
REMOVE Removes the object from the game permanently. msg <ObjectName> REMOVE
CASTSHADOW <bool> Toggles whether the object casts a shadow. msg <ObjectName> (CASTSHADOW 1)<BR>msg <ObjectName> (CASTSHADOW 0)
SETPOS <X> <Y> <Z> DEBUGGING ONLY. Sets the position of an object. msg <ObjectName> (MOVETOPOS 50.5 100 75)
MOVETOPOS <X> <Y> <Z> DEBUGGING ONLY. Moves to the position, with collisions. msg <ObjectName> (MOVETOPOS 50.5 100 75)
SETROTATION <yaw> <pitch> <roll> DEBUGGING ONLY. Rotates object to euler angles. msg <ObjectName> (SETROTATION 90.0 0 0)
COPYXFORM <object> DEBUGGING ONLY. Copies trasform of <object> to this object. msg <ObjectName> (COPYXFORM ThePoint)
ATTACH <object name(s)> Attaches the object specified in the message

If no object is specified it attaches the first object found in the direction of the AttachDir vector or within the dimensions of the world model if AttachDir is <0,0,0>		 To attach a WorldModel named BOX to a WorldModel named BALL, the command would look like:

msg Ball (attach box)
DETACH <object name(s)> Detach a specific object attached with the ATTACH message. To detach a WorldModel named BOX from a WorldModel named BALL, the command would look like:

msg BALL (DETACH BOX)
ATTACHCLASS <class name> This will attach the first object of the specified class found in the direction of AttachDir or within the dimensions of the world model if AttachDir is <0,0,0> To attach any AI to the WorldModel the command would look like:

msg WorldModel (ATTACHCLASS CAI)
DESTROY This command destroys an object in the game as if it had been destroyed trough the loss of all of its hit points. It will play any effects and sounds set up to play at the time of its destruction msg WorldModel DESTROY
NEVERDESTROY <1 or 0> This message allows you to set the NeverDestroy flag through commands. This message does not actually destroy the object but allows destruction if NeverDestroy was set to true initially. msg WorldModel (NEVERDESTROY 1)
ACTIVATE This is an internal message used by game code to specify the WorldModel should activate. This message should not be sent through a commnad specified in the editor. DO NOT USE
RIGIDBODY <bool> This command will set the RigidBody flag on the object making it physically simulated (1) or not (0) msg WorldModel RIGIDBODY 1
SCALEALPHA <0..1> This command will scale the alpha of this world model by the specified value as long as it is flagged as being translucent msg WorldModel ScaleAlpha 0.7

BoxPhysics In the engine, WorldModels have two possible physics models. In the first, the player can walk on and touch every curve and corner of their surface. In the second, the player can only interact with a bounding box that surrounds all the brushes in the WorldModel, just like the box you would get if you selected them in WorldEdit. For geometry with a simple rectangular shape, this is preferred because it's cheaper to calculate. However, for a lot of objects, it's limiting. If you need a player to be able to shoot through the bars in a prison door, you will need BoxPhysics set to FALSE. Boolean 0.000000
IsKeyframed Is the WorldModel moved by a keyframer? If it is you must set this flag to true...or life will end as we know it. Boolean 0.000000
RigidBody Should this world model be simulated with rigid body physics? Boolean 0.000000
MPClientOnlyRigidBody In multiplayer games, should this world model be simulated with rigid body physics only on the client? This flag is ignored if RigidBody is false. Boolean 1.000000
