AINavMeshLinkJump This link is used to specify that the brush contains something the AI must jump down from
AINavMeshLinkJumpOver This link is used to specify that the brush contains something the AI must jump over
AINavMeshLinkLoseTarget If a target goes out of sight while a LoseTarget link falls between the AI and his target, the AI will consider his target lost.
AINavMeshLinkPlayer These links are used to include polys in AIRegions that are not traversable by AI.
AINavMeshLinkStairs This link is used to specify that the brush contains a staircase
AINode This is the most basic node type. The only purpose of a basic AINode is as a destination for a Goto command
AINodeAmbush AI ambush an enemy from an AINodeAmbush
AINodeArmored These nodes are placed out in the open, in good tactical positions to fire from
AINodeCover AI take cover from enemy fire at AINodeCover nodes
AINodeFlipable AI will behave according to the Flipablesmart object
AINodeFollow AI following another character may wait at nodes pointed to by an AINodeFollow, if the leader is within the AINodeFollow's radius or AIRegion
AINodeGoto This node is used as a destination for a Goto command
AINodeGuard This node is the center of an AI's guarding radius. AI will return to an AINodeGuard after giving up a search for the enemy
AINodeInterest This is a node that AI may look at while not engaged in combat.
AINodeIntro When set as an AI's InitialNode in ConfigureAI, the AI walks to this node and plays the animation specified by the SmartObject
AINodeMenacePlace AI will behave according to the MenacePlacesmart object
AINodePatrol Patrolling AI follow a route defines by a chain of AINodePatrol nodes
AINodeScanner This is a special AINodePatrol that defines a place to pause and look around.
AINodeSearch While searching for a threat, AI stop at search nodes and play animations
AINodeStimulus This node defines a stimulus that an AI will sense when coming within range.
AINodeVehicle AI will mount a vehicle from an AINodeVehicle after receiving a MOUNT command
AINodeView An AI will choose to attack from a View nodes when no path can be found to the target and a valid View node is available that points to the area of the level that currently holds the target.
AINodeWorkItem AI will behave according to the WorkItemsmart object
AIRegion AIRegions are used to group portions of an AINavMesh.
AISpawner This object allows the creation of CAI objects. A spawn can be performed using a template object specified in the Target property.
AmmoBox AmmoBoxes are pick up items that can have multiple types of ammo for players to aquire when they walk over an AmmoBox.
Burn Burn volumes define a space where the player recieves Burn damage while within the space.
CAI This is the main AI object that allows specification of which model template to use as well as any specific brain an goals.
Camera This object allows you to setup a cinematic camera.
CameraPoint CameraPoint objects are used in conjunction with Camera objects. You can send a Camera object a MOVETO message to have it move to a specific CameraPoint.
CommandObject The CommandObject is a multitrack command manager that allows for the manipulation and execution of multiple commands on a timeline.
ConstraintHinge Provides a constraint that will join two rigid bodies together at a point and allow for rotation around the forward axis
ConstraintLimitedHinge Provides a constraint that will join two rigid bodies together at a point and allow for rotation around the forward axis
ConstraintPoint Provides a constraint that will join two rigid bodies together at a point
ConstraintPrismatic Provides a constraint that fixes two rigid bodies together, but allows the first rigid body to move along the up axis of the constraint within the range specified.
ConstraintRagdoll Provides a constraint that will join two rigid bodies together at a point and allow for a cone of movement with two cones of restriction, and allow for twist around the forward axis of the object
ConstraintStiffSpring Provides a constraint that will join two rigid bodies together and ensure that the provided points stay a fixed distance away from each other
ConstraintWheel Provides a constraint that fixes two rigid bodies together, but allows the first rigid body to rotate freely around the forward axis of the object. In addition it also allows for suspension along the up axis of the object, allowing the first object to move along the suspension axis within the range specified.
Controller This object enables controlled WorldModels to perform fade functions.
ControlPoint Places a ControlPoint within the level.
CorrosiveFluid CorrosiveFluid volumes are used to define a space with movement and physics properties that behave like water and damage the player while they are within the space.
CTFFlagBase Places a CTFFlagBase within the level.
Dialogue Dialogue objects are used to create conversations between characters.
DynamicSectorVolume Provides a mechanism for disabling visibility through sectors when standing in this volume
Electricity Electricity volumes define a space where the player recieves Electricity damage while within the space.
EndlessFall EndlessFall volumes are used as instant death volumes where the player recieves endless fall damage.
EventCounter The EventCounter object is used to send certain commands for specific values of the event counter. The value of the EventCounter is initialy set through the StartingValue property and then controlled using the INC and DEC messages.
Explosion Explosion objects are used to create a dynamic explosion with a ClientFX, damage type and physics impulse force.
Filter Sound filter volume.
Flicker An object that will generate random intensities convert these into messages that can be sent off to objects for tasks such as flickering lights or world models
GameStartPoint Defines points that the player can start at in a level.
GearItem GearItem objects are used to place Game Database defined Gear records.
Gravity Gravity volumes define a space that can modify the players default gravity while the player is within the volume.
Group Group objects are used to deliver a single message to multiple objects at once. A message received by a group object will be delivered to every object listed within it.
GunMount Allows placement of controllable weapon in level.
Key Key objects are used in conjunction with the KeyFramer object to create KeyFramed paths for objects to follow.
KeyFramer The keyframer is a class that will move the specified objects along a path and can trigger sounds and commands as it moves. The path is specified using either key objects or a path stored in an external key file. The movement rate can either be specified using key times, or can use the distance between keys if TotalPathTime is set to non-zero, in which case it will traverse the full path in that specified amount of time.
KeyframeToRigidBody KeyframeToRigidBody objects are used to turn multiple objects at once to rigid body when it receives the ON message. On the next frame of processing (after a physics update has occured) the command specified will then be sent.
Ladder Binding brushes to this object creates a space that will allow the the player to climb up and down
LightCube A light that projects a cubic environment map out in all directions
LightDirectional A type of light that projects a texture along the specified direction
LightPoint A point light source that emits light equally in all directions
LightPointFill A fast point light source that emits light equally in all directions but does not allow for shadows or specular
LightSpot A type of light that projects a texture along the specified direction
LookAtTrigger Binding brushes to this object creates a space that will execute commands when looked at.
NavMarker A Navigation Marker that will appear on the player's HUD.
NoPlayerTrigger NoPlayerTrigger objects are Triggers that will activate only if no players are within the trigger.
ObjectRemover Remove groups of objects when level starts. If the number of GroupsToKeep is less than the number of groups defined then the groups removed will be chosen and random.
PhysicsImpulseDirectional The PhysicsImpulseDirectional object should be used to apply a physical force the physically simulated objects specifed. These objects will receive ImpulseForce amount of force applied in the direction specified by the foward vector of this PhysicsImpulseDirectional object. The force value is measured in game unit newton seconds.
PhysicsImpulseRadial The PhysicsImpulseRadial object should be used to apply a physical force in a sphere around the position of the object. All physically simulated objects within RadiusMin distance from the PhysicsImpulseRadial object will have ImpulseForce amount of force applied to them in the direction determined by the vector from the center of the PhysicsImpulseRadial object to the center of the physically simulated object. The force applied to physically simulated objects outside of RadiusMin but inside RadiusMax is linearly scaled between ImpulseForce and 0 between RadiusMin and RadiusMax. The force value is measured in game unit newton seconds.
PlayerLure The PlayerLure object is used to properly KeyFrame the Player using the KeyFramer object.
PlayerNodeGoto Specifies a node used for the player message GOTO.
PlayerTrigger PlayerTrigger objects are Triggers that will activate only when all players in a level are within the trigger. Used for multiplayer.
Point Point objects are used solely as position markers in levels. Other game objects and systems refer to point objects only for their transfom information.
PolyGrid Class that simulates the surface of a body of water. If used with a liquid volume brush make sure the top of the volume brush is at the same Y value as the polygrid object if you want bullet vectors to perturb the surface of the water.
PolyGridModifier PolyGridModifiers are used in conjunction with PolyGrids to create disturbances in the PolyGrid object.
PositionalDamage A point light source that emits light equally in all directions

ProximityGrenade An armed proximity grenade

RemoteTurret Places a player controlled remote turret within the level.

RenderTargetGroup An object that will render a scene to a texture and assign that texture to a material for effects such as mirrors or remote cameras

RotatingWorldModel A RotatingWorldModel is a WorldModel object that rotates around a specified point.

ScaleAlpha An object that will allow changing of the alpha scale of associated objects through messages.

ScreenEffect This object adds screen effects to the client, and handles messages for controlling the effects.
SlidingDoor A SlidingDoor is a Door object that moves a specified distance in a specified direction.
SlidingSwitch A SlidingSwitch is a Switch object that moves a specified distance in a specified direction.
SlidingWorldModel A SlidingWorldModel is a WorldModel object that moves a specified distance in a specified direction.
SnowVolume SnowVolumes are used to define a spatial area where snow particles will be created and fall towards the ground.

SoundFX SoundFX objects are used to play a specific sound file at a specific position in the database.
SoundNonPoint SoundNonPoint is used to specify a sound to be played from a non-point source. Examples of it's use are playing water sounds flowing through the length of a pipe.
Spawner This object allows the creation of other objects. A default spawn can be performed using the properties listed in the DefaultSpawn string or an object can be spawned using a template object specified in the Target property.
Speaker Speaker objects are used to play dialogue from non-typical character objects, such as from a telephone or from a speaker system.
SpecialFX This object will play an FXEdit created FX at the position of the object.
SpinningWorldModel SpinningWorldModels are WorldModels that will continualy rotate around a specified point. You can use these to easily set up fans or other continualy rotating objects.
StartupCommand StartupCommand objects are used to send a command immediately when the level loads for the first time.
StoryMode An object to handle turning on and off story mode
SurpriseVolume This object is used with AINodeSurprise objects. This object defines a 'area' for either a near or far attack. See AINodeSurprise property documentation for more usage information.
TeamClientFX Places a TeamClientFX within the level.

VolumeBrush VolumeBrushes allow spaces to be defined that have different movement and physics properties than the default physics and movement.
VolumeEffect VolumeEffect is used solely as the base for other volume based effects.
Water Water volumes are used to define a space with movement and physics properties that behave like water.
WeaponItem A Weapon pickup item that a player can acquire.
Wind Wind volumes define a space where the player will be affected by a wind current.
WorldModel WorldModels are used to specify dynamic geomotry in the world that may be keyframed, have physics properties, be destroyed or interact in other ways with the player or other objects in the world.
WorldProperties This object contains properties, such as fog, that help define the level.
