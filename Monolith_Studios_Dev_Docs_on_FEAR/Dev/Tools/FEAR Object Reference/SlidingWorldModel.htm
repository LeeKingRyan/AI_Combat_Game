SlidingWorldModel
A SlidingWorldModel is a WorldModel object that moves a specified distance in a specified direction.

Message Description Example
VISIBLE <bool> Toggles whether the object is visible. msg <ObjectName> (VISIBLE 1)<BR>msg <ObjectName> (VISIBLE 0)
SOLID <bool> Toggles whether the object is solid. msg <ObjectName> (SOLID 1)<BR>msg <ObjectName> (SOLID 0)
HIDDEN <bool> Toggles whether the object is visible, solid, and rayhit. msg <ObjectName> (HIDDEN 1)<BR>msg <ObjectName> (HIDDEN 0)
REMOVE Removes the object from the game permanently. msg <ObjectName> REMOVE
CASTSHADOW <bool> Toggles whether the object casts a shadow. msg <ObjectName> (CASTSHADOW 1)<BR>msg <ObjectName> (CASTSHADOW 0)
SETPOS <X> <Y> <Z> DEBUGGING ONLY. Sets the position of an object. msg <ObjectName> (MOVETOPOS 50.5 100 75)
MOVETOPOS <X> <Y> <Z> DEBUGGING ONLY. Moves to the position, with collisions. msg <ObjectName> (MOVETOPOS 50.5 100 75)
SETROTATION <yaw> <pitch> <roll> DEBUGGING ONLY. Rotates object to euler angles. msg <ObjectName> (SETROTATION 90.0 0 0)
COPYXFORM <object> DEBUGGING ONLY. Copies trasform of <object> to this object. msg <ObjectName> (COPYXFORM ThePoint)
ATTACH <object name(s)> Attaches the object specified in the message

If no object is specified it attaches the first object found in the direction of the AttachDir vector or within the dimensions of the world model if AttachDir is <0,0,0>		 To attach a WorldModel named BOX to a WorldModel named BALL, the command would look like:

msg Ball (attach box)
DETACH <object name(s)> Detach a specific object attached with the ATTACH message. To detach a WorldModel named BOX from a WorldModel named BALL, the command would look like:

msg BALL (DETACH BOX)
ATTACHCLASS <class name> This will attach the first object of the specified class found in the direction of AttachDir or within the dimensions of the world model if AttachDir is <0,0,0> To attach any AI to the WorldModel the command would look like:

msg WorldModel (ATTACHCLASS CAI)
DESTROY This command destroys an object in the game as if it had been destroyed trough the loss of all of its hit points. It will play any effects and sounds set up to play at the time of its destruction msg WorldModel DESTROY
NEVERDESTROY <1 or 0> This message allows you to set the NeverDestroy flag through commands. This message does not actually destroy the object but allows destruction if NeverDestroy was set to true initially. msg WorldModel (NEVERDESTROY 1)
ACTIVATE This is an internal message used strictly by the game to activate a WorldModel. Do NOT send any WorldModel an ACTIVATE message. DO NOT USE
RIGIDBODY <bool> This command will set the RigidBody flag on the object making it physically simulated (1) or not (0) msg WorldModel RIGIDBODY 1
SCALEALPHA <0..1> This command will scale the alpha of this world model by the specified value as long as it is flagged as being translucent msg WorldModel ScaleAlpha 0.7
TOGGLE Toggles the ON / OFF state of the WorldModel. msg WorldModel01 TOGGLE;
LOCK Locks the WorldModel. A locked WorldModel may not be activated by a message or by the player. msg WorldModel01 LOCK;
UNLOCK Unlocks the WorldModel so it may be activated through messages or by the player. msg WorldModel01 UNLOCK;
PAUSE Pauses the WorldModel while moving or rotating. Sending a PAUSE message to a WorldModel while is is in the ON or OFF state has no effect. Send a RESUME message to continue the movement or rotation. msg WorldModel01 PAUSE;
RESUME Resumes the movement or rotation from a paused WorldModel. Sending a RESUME message to a WorldModel while it is not paused has no effect. msg WorldModel01 RESUME;
SETPOWERONMULTIPLIER <multiplyer> Setting a power on multiplyer will temporairly increase or decrease the speed at which the WorldModel turns on. This is off a base multiplyer of 1 and it will be reset after the first time the WorldModel turns on after setting a multiplyer. Use the ON and OFF with the optional time value for more precise control. msg WorldModel01 (SETPOWERONMULTIPLYER 0.5); Setting the multiplyer to 0.5 will enable the WorldModel to take twice as long to reach the ON state.
CANACTIVATE <1 or 0> Enables or disables the players ability to activate the WorldModel using the action key msg WorldModel01 (CANACTIVATE 0);
SETSTARTINGANGLES <X> <Y> <Z> Allows you to set the WorldModel's StartingAngles via a message msg WorldModel01 (SETSTARTINGANGLES 0 15 0);
ON [time] [waveform] Turn the WorldModel on. The optional parameter [time] is the seconds it takes for the WorldModel to reach the on state. Valid values for the optional parameter [waveform] are: Linear, Sine, SlowOn and SlowOff. To turn the WorldModel on using default values the command would look like:

msg WorldModel01 ON;

To turn a WorldModel on over 2 seconds in a linear movement, the command would look like:

msg WorldModel01 (ON 2.0 Linear);
OFF [time] [waveform] Turn the WorldModel off. The optional parameter [time] is the seconds it takes for the WorldModel to reach the off state. Valid values for the optional parameter [waveform] are: Linear, Sine, SlowOn and SlowOff. To turn the WorldModel off using default values the command would look like:

msg WorldModel01 OFF;

To turn a WorldModel off over half a second in a sine wave movement, the command would look like:

msg WorldModel01 (OFF 0.5 Sine);
ROTATE <X> <Y> <Z> [time] [waveform] Rotate the WorldModel to the specified angle around each axis. All angles are in degrees and relative to the off position of the WorldModel. The optional parameter [time] is the seconds it takes for the WorldModel to reach the angles. Valid values for the optional parameter [waveform] are: Linear, Sine, SlowOn and SlowOff. msg WorldModel01 (ROTATE 0 -15 0 2.5 SlowOn);
MOVE <X> <Y> <Z> [time] [waveform] Move the WorldModel the specified distance along each axis. All distances are in centimeters and relative to the off position of the WorldModel. The optional parameter [time] is the seconds it takes for the WorldModel to move the distance. Valid values for the optional parameter [waveform] are: Linear, Sine, SlowOn and SlowOff. msg WorldModel01 (MOVE -100 50 0 0.5 Linear);

BoxPhysics In the engine, WorldModels have two possible physics models. In the first, the player can walk on and touch every curve and corner of their surface. In the second, the player can only interact with a bounding box that surrounds all the brushes in the WorldModel, just like the box you would get if you selected them in WorldEdit. For geometry with a simple rectangular shape, this is preferred because it's cheaper to calculate. However, for a lot of objects, it's limiting. If you need a player to be able to shoot through the bars in a prison door, you will need BoxPhysics set to FALSE. Boolean 0.000000
