ConstraintPrismatic
Provides a constraint that fixes two rigid bodies together, but allows the first rigid body to move along the up axis of the constraint within the range specified.

Message Description Example
VISIBLE <bool> Toggles whether the object is visible. msg <ObjectName> (VISIBLE 1)<BR>msg <ObjectName> (VISIBLE 0)
SOLID <bool> Toggles whether the object is solid. msg <ObjectName> (SOLID 1)<BR>msg <ObjectName> (SOLID 0)
HIDDEN <bool> Toggles whether the object is visible, solid, and rayhit. msg <ObjectName> (HIDDEN 1)<BR>msg <ObjectName> (HIDDEN 0)
REMOVE Removes the object from the game permanently. msg <ObjectName> REMOVE
CASTSHADOW <bool> Toggles whether the object casts a shadow. msg <ObjectName> (CASTSHADOW 1)<BR>msg <ObjectName> (CASTSHADOW 0)
SETPOS <X> <Y> <Z> DEBUGGING ONLY. Sets the position of an object. msg <ObjectName> (MOVETOPOS 50.5 100 75)
MOVETOPOS <X> <Y> <Z> DEBUGGING ONLY. Moves to the position, with collisions. msg <ObjectName> (MOVETOPOS 50.5 100 75)
SETROTATION <yaw> <pitch> <roll> DEBUGGING ONLY. Rotates object to euler angles. msg <ObjectName> (SETROTATION 90.0 0 0)
COPYXFORM <object> DEBUGGING ONLY. Copies trasform of <object> to this object. msg <ObjectName> (COPYXFORM ThePoint)

Property Description Type Default
Name The name of this object String noname

Rotation The orientation of this object represented by euler angles in degrees Rotation 0.000000, 0.000000, 0.000000

ObjectLOD This value indicates at which detail levels the object will exist. For example at Low, this object will always exist, at Medium it will only exist in medium and high detail, and Never will cause the object to never exist. String (StaticList) Low
Object1 The name of the first object that this constraint will link. This should be left blank if it is intended to link to the world. String
ModelNode1 The name of the node within the first object that the constraint will link to. This should be left blank if the object is not a model. String
Object2 The name of the second object that this constraint will link. This should be left blank if it is intended to link to the world. String
ModelNode2 The name of the node within the second object that the constraint will link to. This should be left blank if the object is not a model. String
Breakable Determines if this constraint can be broken at runtime when a force that exceeds the threshold is applied Boolean 0.000000
BreakForce Indicates the force that when exceeded, the constraint will be broken. This number is largely determined through experimentation, but higher numbers mean more force is required to break the constraint Float 100.000000
StartBroken Determines if the constraint should start broken or not. Only applicable to breakable constraints Boolean 0.000000
BreakCmd Command that will be triggered when this breakable constraint is broken. Command
MPClientOnly In multiplayer games the constraints need to be created on the client. This flag is ignored in singleplayer games. Boolean 1.000000
MovementMin The lower range of movement along the up axis of this constraint. This should be less than zero and less than the movement maximum. Float 0.000000
MovementMax The upper range of movement along the up axis of this constraint. This should be more than zero and more than the movement minimum. Float 0.000000
Friction The amount of friction to apply as the rigid body moves along the axis to slow down the movement. Higher values slow the movement down more. Float 0.000000
